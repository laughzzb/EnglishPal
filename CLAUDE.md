## 心理陪伴小程序 — 产品蓝图

### 核心理念
> **「了解一个人，就是要知道他的很多事情、很多想法，才会形成一个人。」**

不是做「AI 心理聊天」，而是做「一个了解你的 AI」。两者区别：
- AI 心理聊天：每次从零开始，问题单独处理，用完即走
- 了解你的 AI：带着用户档案来，链接前后问题，时间越长越懂你

### 产品价值观（用户原话）
- 「只有当人的情绪被调动，才说明这样的对话是有意义的」
- 「不能用表面回复判断价值，要综合用户行为和状态判断」
- 「哪怕一次看不出来也没有关系」——心理陪伴是缓慢的信任建设
- 「自信到我不怕你拒绝」——隐私告知放在明处，不偷偷摸摸
- 「勇气开启新的篇章同样是一种新的开始」——删除数据的确认文案

### MVP 目标用户（四层）
1. 中年危机的中年人
2. **刚毕业就业焦虑的年轻人（MVP 首选）**
3. 高考前高压的考生
4. 青春期的青少年

MVP 首选毕业焦虑群体，因为这是用户自身的处境，最能共情。

### 核心洞察
> **「时间不会等你想清楚再往前走，绿灯亮了你就得选一条路。」**

中国式教育（愧欠式教育、绑架式教育、应试教育）导致很多年轻人在毕业时：
- 刚刚意识到自己是独立个体
- 还没搞清楚自己是谁、想要什么
- 就被迫做出人生选择

### 核心功能流程
```
用户进入 → 性格测试（8-10题，荣格类型） → 人格分类 → 引导式对话 → 总结梳理
                                                                  ↓
                                                           保存心理档案
                                                                  ↓
                                                   下次打开 → 载入档案 → 接着聊
```

### 引导式对话哲学（苏格拉底式提问）
AI 核心任务不是「给答案」，而是「理问题」：
- 你真正纠结的是什么？
- 你试过什么方法？效果如何？
- 如果现在有一个机会改变一件事，你选什么？

### prompt 构建方法
用户收集素材（抖音评论、文章、书籍等）→ 交给助手解构 → 拆成判断逻辑/问题结构 → 写成 prompt 规则 → 喂进小程序

不是搬运台词，是消化成对话判断逻辑。

### 心理档案核心字段
- 测试分类结果
- 每次对话的核心议题
- 用户表达习惯（喜欢建议还是被问问题）
- 哪些问题被解决 / 哪些还在悬着

### 危机干预三级机制
1. **想法层面：** AI 判断为想法但未行为化 → 给心理热线号码 + 提示
2. **行为/计划层面：** 用户已做出现实中行为（买药、自残等）→ 弹出紧急联系电话（120 等），不再分析不再安慰
3. 界面可感知敏感词汇，改变界面提示用户需要专业治疗
4. 核心原则：**不能、也不可以替代心理治疗。产品定位是陪伴，不是治疗**

### 隐私数据处理
- 首次使用时告知：测试结果用于个性化对话，对话记录会被保存以更了解你
- 告知时机：测试完成后、对话开始前（不是对话结束后弹窗）
- 用户可自行删除数据，且提供双重确认
- 数据只有用户自己可见

### 技术路线
- 平台：**微信小程序（先不做 App）**
- AI：调用 DeepSeek API，不做模型微调
- 核心工作：Prompt 工程（对话节奏控制 + 分类策略 + 引导逻辑）
- 启动预算：约 300 元
- 预期周期：约 1 周出 MVP

### 商业模式思考（初步）
- 核心诉求：通过产品实现价值、帮助人，同时有被动收入解决吃饭问题
- 不急于收费，可接广告或设置低门槛付费
- 基础免费 + 增值模式待定
- 推广渠道：公众号推文 + 小红书种草文案（助手可协助）
- 合规红线：不能自称 AI 心理医生，不能诊断抑郁症，不能碰危机干预

### 要解决的市场痛点
- 现有 AI（文心一言、DeepSeek 等）不够垂直、不够细腻、无法共情
- 绝大多数中国人接触 AI 的门槛仍然很高
- 心理咨询市场供不应求（40 万人只有一个精神卫生专业人士），收费高

## 工作方式约定

- **用户不主动让读取文件就不读**，减少不必要的文件读取以避免上下文丢失
- EnglishPal 是主线，用户自己把控进度，助手不催促
- 用户收集素材 → 传给助手 → 助手拆解为 prompt 规则 → 反馈给用户
- 用户与助手不断对话磨产品想法，确保做出真正有用的东西
